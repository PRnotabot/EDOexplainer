Shape Optimization with the Adjoint Method

From NURBS Control Points to Optimal Geometries

Based on Engineering Design Optimization by Martins & Ning

The Motivation

Shape Determines Performance

Small changes in geometry can cause large changes in aerodynamic drag, structural stress, or heat transfer.

Engineers have always known this. The question is: how do we find the best shape systematically?

Design optimization replaces trial-and-error with mathematics.

The Optimization Problem

Every design optimization has the same structure:

minimize   f(x)

subject to   g(x) ≤ 0

                xL ≤ x ≤ xU

Design Variables x

Control point positions that define the shape

Objective f(x)

Drag, weight, compliance — what we minimize

Constraints g(x)

Stress limits, volume, lift requirements

How Do We Describe a Shape Mathematically?

We need a parametrization — a way to define any shape using a finite set of numbers.

Too Few Parameters

Limited design freedom. Might miss the optimal shape entirely.

Too Many Parameters

Expensive optimization. Difficult to converge.

NURBS (Non-Uniform Rational B-Splines) give us smooth, flexible shapes with relatively few control parameters.

Shape Parametrization

NURBS Curves

Control points define the shape — but the curve does not pass through them.

The control polygon (connecting the control points) approximates the curve's shape.

C(t) = Σ Ni,p(t) · Pi

where Ni,p are B-spline basis functions and Pi are control points

B-Spline Basis Functions

Each control point Pi has a corresponding basis function Ni,p(t) that determines its region of influence.

Moving a control point only affects the curve locally — not the entire shape.

This local support property is why NURBS are ideal for shape optimization — each design variable has a bounded effect.

Try It: Drag the Control Points

Drag the cyan points to reshape the curve. Notice how each point only affects the nearby portion.

We can describe any shape with NURBS.

Now, how do we find the best shape?

Gradient-Based Optimization

The Optimization Loop

Search Direction

Steepest Descent

The gradient ∇f points in the direction of steepest increase.

So we move in the opposite direction:

p = −∇f

Simple and intuitive — but it has a problem.

Watch Gradient Descent Navigate

Click anywhere to set a starting point. Observe the zigzag path of steepest descent on elongated contours.

Smarter Search Directions

Steepest descent ignores curvature. Better methods use it.

Conjugate Gradient

Builds search directions that are conjugate to previous ones. Avoids revisiting the same directions.

Quasi-Newton (BFGS)

Approximates the Hessian (curvature matrix) from gradient history. Converges superlinearly.

BFGS is the workhorse of gradient-based shape optimization.

Constrained Optimization

Real problems have constraints — stress limits, volume, lift requirements.

Sequential Quadratic Programming (SQP) solves a sequence of quadratic subproblems, each respecting the linearized constraints.

At the optimum, active constraints are satisfied exactly: g(x*) = 0. Inactive constraints don't affect the solution.

Every Method Needs Gradients

Steepest descent, conjugate gradient, BFGS, SQP — they all require:

∇f = [ ∂f/∂x1, ∂f/∂x2, ..., ∂f/∂xn ]

In shape optimization, n can be hundreds or thousands of design variables.

How do we compute these gradients efficiently?

The optimizer knows how to navigate.

But it needs a compass — the gradient.

Computing this compass is the real challenge.

Method 1

Finite Differences

The simplest approach: perturb each variable and re-evaluate.

∂f/∂xi ≈ [f(x + hêi) − f(x)] / h

Problem: needs one function evaluation per design variable.

500 design variables = 501 physics solves

The Physics

The Implicit Model

Engineering analysis involves solving governing equations:

r(u; x) = 0

x = design variables (control points)
u = state variables (pressure, velocity, displacement)
r = residual equations (conservation laws, equilibrium)

The objective f(x, u) depends on x both directly and through u.

Total Derivative via Chain Rule

Because f depends on x both directly and through u:

df/dx = ∂f/∂x + (∂f/∂u)(du/dx)

∂f/∂x

Direct effect. Cheap to compute.

∂f/∂u

Sensitivity of f to states. Also cheap.

du/dx

How states change with design. Expensive!

The challenge is computing du/dx without re-solving the nonlinear equations nx times.

Differentiating the Governing Equations

Since r(u; x) = 0 must hold, differentiating with respect to x gives:

(∂r/∂u) (du/dx) = −(∂r/∂x)

∂r/∂u

Square matrix (nu × nu). We already have this from the solver!

∂r/∂x

Matrix (nu × nx). Cheap partial derivatives.

A linear system — much cheaper than re-solving the original nonlinear equations!

Forward Mode

The Direct Method

Solve the linear system once per design variable:

(∂r/∂u) φi = ∂r/∂xi

for i = 1, 2, ..., nx

Then compute the total derivative:

df/dx = ∂f/∂x − (∂f/∂u) φ

Cost: nx linear solves
For 500 design variables = 500 solves

The Key Insight

The Adjoint Method

Instead of solving column by column (one per design variable)...

...solve row by row (one per function of interest):

(∂r/∂u)T ψ = (∂f/∂u)T

Then compute all derivatives at once:

df/dx = ∂f/∂x − ψT (∂r/∂x)

Cost: nf linear solves
For 1 objective = 1 solve (regardless of number of design variables!)

One Solve → All Gradients

Since nx >> nf, the adjoint method is orders of magnitude cheaper than finite differences or the direct method.

"One widespread application of the adjoint method has been in aerodynamic and hydrodynamic shape optimization."

Martins & Ning, Engineering Design Optimization, 2022

The adjoint method makes shape optimization computationally feasible.

Key Takeaways

  • Shape parametrization matters. NURBS give smooth, flexible geometry with local control — ideal for optimization.
  • Gradient-based methods are essential for high-dimensional design spaces. BFGS and SQP are the workhorses.
  • The adjoint method is the key enabler. It computes all gradients at a cost independent of the number of design variables.
  • Cost scales with functions, not variables. 1 objective + few constraints = very few adjoint solves, regardless of shape complexity.
  • This is state of the art. NURBS + adjoint + gradient-based optimization powers design in aerospace, automotive, and beyond.
